# SECURITY.md - Política de Segurança / Security Policy

## PT-BR / Português

### Relatando uma Vulnerabilidade

Para relatar uma vulnerabilidade de segurança, utilize o recurso de [relatório privado de vulnerabilidades do GitHub](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability).

### Notas de Segurança

- Nunca commite arquivos `.env` ou credenciais.
- Mantenha configurações sensíveis apenas no ambiente local.
- Utilize variáveis de ambiente para segredos.

---

## EN / English

### Reporting a Vulnerability

To report a security vulnerability, use [GitHub's private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability) feature.

### Security Notes

- Never commit `.env` files or credentials.
- Keep sensitive configuration local only.
- Use environment variables for secrets.
